    Harneet Error Handling System Guide

    üéâ NEW: Go-Style Error Handling Implemented!

    Harneet now features a comprehensive Go-style error handling system that prevents silent failures and requires explicit error checking. This is a BREAKING CHANGE that makes Harneet programs more reliable and predictable.

    ‚úÖ Key Features

    1. Multiple Return Values: Functions return (result, error) tuples
    2. Explicit Error Checking: Must check if err != None
    3. No Silent Failures: All errors must be handled explicitly
    4. Comprehensive Coverage: All standard library functions now return errors
    5. None Values: None represents successful operations (no error)

    üìö Basic Syntax

    Multiple Return Values

    // All stdlib functions now return (result, error) tuples
var result, err = math.Max(10, 20)
var length, lenErr = strings.Len("hello")
var number, parseErr = strings.ToInt("42")

    Error Checking Pattern

    var result, err = someFunction()
if err != None {
    // Handle the error
    fmt.Println("Error occurred:", err)
    // Could return, exit, or continue with alternative logic
} else {
    // Use the result safely
    fmt.Println("Success:", result)
}

    üîÑ Migration Examples

    Before (Old Syntax - BROKEN)

    // This will now cause errors!
var result = math.Max(10, 20)              // ‚ùå Wrong
var length = strings.Len("hello")          // ‚ùå Wrong  
var maxVal = math.Max(math.Abs(-5), 10)    // ‚ùå Wrong

    After (New Syntax - CORRECT)

    // Correct error handling syntax
var result, err = math.Max(10, 20)         // ‚úÖ Correct
if err != None {
    fmt.Println("Error:", err)
} else {
    fmt.Println("Result:", result)
}

// Chained operations with error handling
var absResult, absErr = math.Abs(-5)
if absErr != None {
    fmt.Println("Error in abs:", absErr)  
} else {
    var maxResult, maxErr = math.Max(absResult, 10)
    if maxErr != None {
        fmt.Println("Error in max:", maxErr)
    } else {
        fmt.Println("Final result:", maxResult)
    }
}

    üèõÔ∏è Error Handling Patterns

    Pattern 1: Simple Error Check

    var result, err = math.Abs(-25)
if err != None {
    fmt.Println("Error:", err)
} else {
    fmt.Println("Success:", result)
}

    Pattern 2: Early Return (Simulation)

    var number, err = strings.ToInt("invalid")
if err != None {
    fmt.Println("Conversion failed:", err)
    // In a real function, you would return here
    fmt.Println("Would return early in a function")
} else {
    fmt.Println("Continuing with:", number)
}

    Pattern 3: Multiple Error Sources

    var num1, err1 = strings.ToInt("25")
var num2, err2 = strings.ToInt("30")

if err1 != None {
    fmt.Println("First conversion failed:", err1)
} else if err2 != None {
    fmt.Println("Second conversion failed:", err2)
} else {
    // Both conversions succeeded
    var sum int = num1 + num2
    fmt.Println("Sum:", sum)
}

    Pattern 4: Error Propagation Chain

    // Convert string to number, then find absolute value, then find max
var numStr string = "-42"
var num, numErr = strings.ToInt(numStr)
if numErr != None {
    fmt.Println("Number conversion failed:", numErr)
} else {
    var abs, absErr = math.Abs(num)
    if absErr != None {
        fmt.Println("Absolute value failed:", absErr)
    } else {
        var max, maxErr = math.Max(abs, 50)
        if maxErr != None {
            fmt.Println("Max calculation failed:", maxErr)
        } else {
            fmt.Println("Final result:", max)
        }
    }
}

    üì¶ Updated Function Signatures

    Math Module

    Old Signature New Signature
    math.Abs(x) ‚Üí int math.Abs(x) ‚Üí (int, error)
    math.Max(x, y) ‚Üí int math.Max(x, y) ‚Üí (int, error)
    math.Min(x, y) ‚Üí int math.Min(x, y) ‚Üí (int, error)
    math.Pow(base, exp) ‚Üí int math.Pow(base, exp) ‚Üí (int, error)

    Strings Module

    Old Signature New Signature
    strings.Len(s) ‚Üí int strings.Len(s) ‚Üí (int, error)
    strings.Upper(s) ‚Üí string strings.Upper(s) ‚Üí (string, error)
    strings.Lower(s) ‚Üí string strings.Lower(s) ‚Üí (string, error)
    strings.Contains(s, sub) ‚Üí int strings.Contains(s, sub) ‚Üí (int, error)
    strings.Replace(s, old, new) ‚Üí string strings.Replace(s, old, new) ‚Üí (string, error)
    strings.Split(s, sep) ‚Üí string strings.Split(s, sep) ‚Üí (string, error)
    strings.ToInt(s) ‚Üí int strings.ToInt(s) ‚Üí (int, error)

    Other Modules

    All standard library modules (datetime, os, path, file, log) now follow the same pattern of returning (result, error) tuples.

    üß™ Testing Error Conditions

    Valid Operations (No Errors)

    var result, err = math.Max(10, 20)
// result = 20, err = None

var length, lenErr = strings.Len("hello")  
// length = 5, lenErr = None

var number, parseErr = strings.ToInt("42")
// number = 42, parseErr = None

    Invalid Operations (With Errors)

    var number, err = strings.ToInt("not_a_number")
// number = None, err = "cannot convert 'not_a_number' to integer"

// Note: Math operations typically don't error in current implementation
// but the infrastructure is there for future enhancements

    ‚ö° Benefits

    1. üõ°Ô∏è No Silent Failures: Errors can't be ignored
    2. üîç Clear Error Messages: Descriptive error information
    3. üèóÔ∏è Reliable Code: Forces proper error handling
    4. üìç Error Context: Know exactly what went wrong where
    5. üöÄ Go-Style Patterns: Familiar to Go developers

    üîß How to Update Existing Code

    Step 1: Identify Function Calls

    Find all calls to standard library functions: - math.* functions - strings.* functions
    - datetime.* functions - os.* functions - path.* functions - file.* functions - log.* functions

    Step 2: Update to Multiple Assignment

    // Before
var result = math.Max(a, b)

// After  
var result, err = math.Max(a, b)
if err != None {
    // Handle error
} else {
    // Use result
}

    Step 3: Handle Chained Operations

    // Before (BROKEN)
var result = math.Max(math.Abs(x), y)

// After (CORRECT)
var absX, absErr = math.Abs(x)
if absErr != None {
    // Handle abs error
} else {
    var result, maxErr = math.Max(absX, y)
    if maxErr != None {
        // Handle max error  
    } else {
        // Use result
    }
}

    üö® Breaking Changes Summary

    • ‚ùå All old code using stdlib functions will break
    • ‚úÖ Must update to use var result, err = function() syntax
    • ‚úÖ Must add if err != None checks
    • ‚úÖ Benefits: Much more reliable and predictable code

    This error handling system makes Harneet programs more robust by forcing developers to explicitly handle all potential error conditions, preventing unexpected crashes and silent failures.
